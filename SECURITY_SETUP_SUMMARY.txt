## 🎯 Для команды разработки

### Новый разработчик должен:

```bash
# 1. Клонировать репо
git clone <репо>

# 2. Скопировать примеры
cp .env.example .env
cd server && cp .env.example .env && cd ..

# 3. Попросить реальные ключи у тимлида
# (Они НЕ в репо для безопасности!)

# 4. Вставить реальные значения
nano .env
nano server/.env

# 5. Начать разработку
npm install && cd server && npm install && cd ..
npm run dev
# + в другом терминале: cd server && npm run dev
```

### Если .env был запушен:

```bash

git commit -m "Emergency: Remove .env with secrets"
git push

# 2. Регенерируем все ключи Supabase/БД
# (Старые ключи больше НЕ работают!)

# 3. Обновляем новые ключи локально
cp .env.example .env
# вставляем новые значения
```
