# 🔐 Registration Guide - Chain Scout Web

## Почему я не могу зарегистрироваться?

Как вы правильно упомянули, **проект еще не захостен**, поэтому регистрация работает только **локально** для разработки.

Это **обычное явление** - в разработке нужны оба сервера (фронтенд и бэкенд) для полного функционала.

## ✅ Быстрая регистрация (5 минут)

### Шаг 1: Установить зависимости

**Вариант A - Автоматический (рекомендуется):**
```powershell
# Windows PowerShell
.\setup.ps1
```

**Вариант B - Ручной:**
```bash
npm install
cd server && npm install && cd ..
```

### Шаг 2: Запустить оба сервера

**Терминал 1 - Фронтенд:**
```bash
npm run dev
```
*(Откроется на http://localhost:5173)*

**Терминал 2 - Бэкенд:**
```bash
cd server
npm run dev
```
*(Запустится на http://localhost:4000)*

### Шаг 3: Зарегистрироваться

1. Откройте http://localhost:5173
2. Нажмите "Sign Up" 
3. Введите email и пароль
4. Нажмите "Register"

### Шаг 4: Проверить почту

1. **Откройте вашу почту** (Gmail, Outlook и т.д.)
2. **Найдите письмо** от `noreply@rrcbgqledrotnnvpjtnk.supabase.co`
3. **⚠️ ВАЖНО:** Проверьте папку **Спама/Junk** - письио может там быть!
4. **Нажмите** на ссылку в письме
5. **Вернитесь** в приложение и войдите

## 🆘 Если не работает - Решение проблем

### Проблема: "Email не приходит"

**Решение:**
- [ ] Проверьте **папку спама** (очень часто там!)
- [ ] Проверьте правильность email (нет опечаток?)
- [ ] Подождите **2-3 минуты** (письмо может идти медленно)
- [ ] Попробуйте **другой email**
- [ ] Убедитесь что **оба сервера запущены**
  ```bash
  # Terminal 1
  npm run dev
  
  # Terminal 2
  cd server && npm run dev
  ```

### Проблема: "Ошибка при регистрации"

**Проверьте консоль браузера (F12):**
1. Нажмите **F12** в браузере
2. Откройте вкладку **Console**
3. Посмотрите на **красные ошибки**
4. Скопируйте текст ошибки

**Возможные ошибки:**
```
Error: Missing Supabase environment variables
→ Решение: Проверьте файл .env
```

```
Connection refused on http://localhost:4000
→ Решение: Запустите бэкенд сервер (cd server && npm run dev)
```

```
CORS error
→ Решение: Убедитесь что CORS настроен в server/index.js
```

### Проблема: "ECONNRESET при npm install"

**Это сетевая проблема**, попробуйте:
```bash
npm cache clean --force
npm install
```

Или используйте альтернативный npm registry:
```bash
npm install -g npm@latest
npm install
```

## 📊 Проверочный список

Перед тем как делать регистрацию, убедитесь:

- [ ] **Node.js установлен**: `node -v` (должно показать версию)
- [ ] **npm установлен**: `npm -v` (должно показать версию)
- [ ] **Зависимости установлены**: в папке есть `node_modules` folder
- [ ] **Фронтенд запущен**: http://localhost:5173 открывается
- [ ] **Бэкенд запущен**: http://localhost:4000 можно посетить
- [ ] **Файл .env есть**: содержит `VITE_SUPABASE_*` переменные
- [ ] **Интернет подключен**: проверьте connection к Supabase
